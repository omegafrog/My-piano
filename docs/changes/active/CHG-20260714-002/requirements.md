# 요구사항

## 목표

- 운영자가 사용자에게 할인 쿠폰을 수동 발급하고, 사용자가 주문 결제에 적용할 수 있게 한다.

## 행위자

- 운영자: 대상 사용자에게 쿠폰을 수동 발급한다.
- 사용자: 보유 쿠폰을 주문 결제에 적용한다.

## 범위

- 포함:
  - 운영자의 사용자 대상 쿠폰 수동 발급
  - 정액 할인과 정률 할인 쿠폰
  - 최소 주문금액, 최대 할인액, 유효기간, 1회 사용 제한
  - 결제 직전 쿠폰 임시 적용
  - 결제 성공 시 쿠폰 사용 처리
  - 결제 실패 시 임시 적용 복구
- 제외:
  - 자동·대량 쿠폰 발급
  - 쿠폰 정책 외의 결제 수단 또는 주문 정책 변경

## 기능 요구사항

### FR-001

- 요구사항: 운영자는 특정 사용자에게 할인 쿠폰을 수동 발급할 수 있다.

### FR-002

- 요구사항: 쿠폰은 정액 할인 또는 정률 할인을 지원하고, 최소 주문금액, 최대 할인액, 유효기간, 1회 사용 가능 여부를 가진다.

### FR-003

- 요구사항: 사용자는 유효하고 미사용이며 주문금액 조건을 충족하는 보유 쿠폰을 주문 결제에 적용할 수 있다.

### FR-004

- 요구사항: 쿠폰은 결제 직전에 임시 적용되며, 결제 성공 시 1회 사용 처리된다.

### FR-005

- 요구사항: 임시 적용 뒤 결제가 실패하면 쿠폰 적용을 복구하여 다시 사용할 수 있게 한다.

## 성공/실패 기준

- 성공:
  - 운영자가 사용자에게 쿠폰을 발급할 수 있다.
  - 조건을 충족한 쿠폰 할인액이 주문 결제에 반영된다.
  - 결제 성공 후 해당 쿠폰은 다시 사용할 수 없다.
  - 결제 실패 후 해당 쿠폰은 다시 사용할 수 있다.
- 실패:
  - 유효기간 만료, 이미 사용, 최소 주문금액 미달 쿠폰은 적용되지 않는다.
  - 정률 할인 결과가 최대 할인액을 초과하면 최대 할인액까지만 할인된다.

## 비기능 요구사항

### NFR-001

- 요구사항: 결제 성공·실패 처리에서 쿠폰 사용 가능 상태가 결과와 일관되게 유지되어야 한다.
