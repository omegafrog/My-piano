# 사용자 흐름

## 커뮤니티 콘텐츠 둘러보기

1. 사용자가 커뮤니티 게시글 목록을 요청한다.
2. 백엔드는 로컬 데이터베이스에 저장된 게시글과 작성자 정보를 반환한다.
3. 사용자가 게시글을 선택하면 상세 API가 본문과 관련 정보를 반환한다.
4. 화면은 응답에 포함된 `/profiles/**` 프로필 이미지를 백엔드에서 불러온다.

로컬 초기 콘텐츠에는 사용자 5명, 커뮤니티 게시글 8개와 댓글이 포함된다. 제목과 본문은 자연스러운 한국어 콘텐츠로 구성되며 로컬 검증에서 금칙어 노출 여부를 함께 검사한다.

## 악보 둘러보기

1. 사용자가 악보 목록을 요청한다.
2. 백엔드는 악보 6개와 작성자, 썸네일 정보를 반환한다.
3. 사용자가 악보를 선택하면 상세 API가 악보 정보를 반환한다.
4. 화면은 `/thumbnails/**`에서 썸네일을, `/sheets/**`에서 PDF 악보를 불러온다.

로컬 자산은 프로필 SVG 5개, 썸네일 SVG 6개, 유효한 PDF 6개로 구성된다. 모든 자산 URL은 백엔드의 정적 자원 경로를 사용한다.

## 프런트엔드 연동 상태

프런트엔드는 Node.js 20 환경에서 의존성 설치, 빌드, 기동과 3000 포트 응답까지 확인됐다. 다만 API 경로 접두사가 빠진 요청으로 CORS 오류가 발생해 백엔드 API 연동은 완료되지 않았다. 재현 정보와 백엔드 독립 성공 근거는 [프런트엔드 Issue #3](https://github.com/omegafrog/my-piano-frontend/issues/3)에 기록돼 있다.

이 문제는 백엔드 로컬 배포 성공 여부와 분리된다. 백엔드 기능과 초기 콘텐츠는 [검증](verification.md)의 독립 smoke 검증으로 확인한다.
